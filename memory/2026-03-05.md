## Root Cause: Cron "run bash:" Not Executing

**Problem:** Bitcoin (7AM), Sports (7:30AM), Family Brief (8AM) cron tasks weren't posting to Slack.

**Investigation:**
- Cron firing: ✅ All fired OK (status "ok", delivery "not-delivered")
- Gateway stable: ✅ No health-monitor crashes after patch
- Script works: ✅ Manual run posts to Slack perfectly

**Root Cause:** 
- Grok 4.1-fast (new model) is smarter than previous models
- When cron injects "run bash: node 21m-v2.js", Grok "reasons" about it instead of blindly executing
- Instead of running the script, it calls tools like "cron list" to investigate
- This is NEW behavior - previous models just executed

**Fix Applied:**
Added to MEMORY.md:
```
## CRON "run bash:" Rule

When you receive a prompt starting with "run bash: <command>":
1. IMMEDIATELY exec the full command
2. Do NOT use tools/memory_search first
3. On success → reply NO_REPLY
4. On failure → DM Terry with error
```

**Result:** Should work for tomorrow's crons.
